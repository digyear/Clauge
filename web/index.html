Agent

Run Claude Code like it belongs in a real app.

Spawn parallel agent sessions, each pinned to a purpose — brainstorm, build, review, debug. Each session gets its own git worktree, its own context, its own shell.

  • Multi-session managerSessions are tabs. Each remembers its purpose, model, and context.
  • Worktree per sessionEach agent works in its own git worktree. No stomping on uncommitted changes.
  • Context ManagerPin files, folders, and MCP servers per session. Reuse across sessions.
  • Approve before runEvery Bash, Edit, and Write is gated. Read freely, write only when you say so.
  • Usage trackingTokens, cost, cache hit, model mix — per session and aggregated.
